What is a Compiler?

A compiler is a program that translates code written in one programming language into another language, usually machine code that computers can execute.

Simple Explanation: Think of a compiler as a translator that converts your Python or C code into the language the computer's processor understands.

Compiler vs Interpreter

  • Compiler: Translates the entire program at once, creates an executable file
  • Interpreter: Translates and runs code line by line

Stages of Compilation

  1. Reading: The compiler reads your source code
  2. Checking: It checks for errors in your code
  3. Translating: It converts to machine language
  4. Creating: It produces an executable program

Output Tokens:

Syntax Analysis (Parser)

Creates an Abstract Syntax Tree (AST) using grammar rules:

Grammar (BNF):
expr   → term (('+' | '-') term)*
term   → factor (('*' | '/') factor)*
factor → NUMBER | IDENTIFIER | '(' expr ')'

Advanced Compiler Design

Deep dive into optimization techniques, code generation, and advanced parsing.

Parser Types

  • Top-Down (LL): Recursive descent, LL(k) parsers
  • Bottom-Up (LR): SLR, LALR, CLR parsers

Intermediate Representations

# Three-Address Code (TAC)
# Source: a = b * c + d * e

t1 = b * c
t2 = d * e
t3 = t1 + t2
a = t3

# Static Single Assignment (SSA)
t1₁ = b₀ * c₀
t2₁ = d₀ * e₀
t3₁ = t1₁ + t2₁
a₁ = t3₁

Optimization Techniques

1. Constant Folding:
   x = 3 + 5  →  x = 8

2. Dead Code Elimination:
   x = 5
   x = 10    ← Remove first assignment
   print(x)

3. Loop Invariant Code Motion:
   for i in range(n):
       x = a + b        ← Move outside loop
       arr[i] = x * i

   x = a + b
   for i in range(n):
       arr[i] = x * i

4. Common Subexpression Elimination:
   a = b + c
   d = b + c  →  d = a

Register Allocation

Graph coloring algorithm for register assignment:

  • Build interference graph (variables as nodes)
  • Simplify: remove nodes with degree < k
  • Spill: if stuck, spill a variable to memory
  • Select: assign colors (registers) to nodes

Code Generation

# TAC to x86-64 Assembly
# t1 = a + b

mov  rax, [a]      ; Load a into rax
add  rax, [b]      ; Add b to rax
mov  [t1], rax     ; Store result in t1
