What is Theory of Computation?

Theory of Computation studies what problems computers can solve and how efficiently they can solve them. It's the mathematical foundation of computer science.

Simple Explanation: Just like math has rules, computers have rules about what they can and can't do. This field studies those rules.

Three Main Questions

  1. What is computation? Formal models of computing
  2. What can be computed? Decidability and computability
  3. How efficiently? Complexity theory

Simple Machines

We use abstract machines to understand computation:

  • Finite Automata: Like a vending machine with states
  • Pushdown Automata: Can remember things in a stack
  • Turing Machine: Can solve any computable problem

Automata Theory

Understanding formal languages and the machines that recognize them.

Chomsky Hierarchy

Type 0: Recursively Enumerable → Turing Machine
Type 1: Context-Sensitive      → Linear Bounded Automaton
Type 2: Context-Free           → Pushdown Automaton
Type 3: Regular                → Finite Automaton

Finite Automata

DFA for strings ending in "01":
States: {q0, q1, q2}
Alphabet: {0, 1}
Start: q0
Accept: {q2}

Transitions:
     0    1
q0   q1   q0
q1   q1   q2
q2   q1   q0

Example: "1101" → q0→q0→q0→q1→q2 ✓

Regular Expressions

Equivalent to finite automata:

Pattern: (0|1)*01
Matches: "01", "101", "0001", "1101"

Regex operators:
* (Kleene star): zero or more
+ (plus): one or more
? (optional): zero or one
| (union): either/or

Context-Free Grammars

Grammar for balanced parentheses:
S → (S) | SS | ε

Derivation of "(())":
S → (S) → ((S)) → (())

Advanced Computability and Complexity

Exploring undecidability, complexity classes, and the limits of computation.

The Halting Problem

Proven undecidable by Alan Turing using diagonalization:

Theorem: There is no algorithm that can determine
whether an arbitrary program will halt on a given input.

Proof by contradiction:
1. Assume HALT(P, I) exists, returns true/false
2. Construct D(P):
   if HALT(P, P) then loop_forever()
   else return true
3. What is D(D)?
   - If HALT(D, D)=true → D loops → contradiction
   - If HALT(D, D)=false → D halts → contradiction
∴ HALT cannot exist

Complexity Classes

P:     Polynomial time (efficiently solvable)
       Examples: Sorting, MST, Shortest Path

NP:    Nondeterministic Polynomial (verifiable)
       Examples: SAT, Clique, Vertex Cover

NP-Complete: Hardest in NP
       - In NP
       - Every NP problem reduces to it

NP-Hard: At least as hard as NP-Complete
       May not be in NP

Open Question: P = NP?

Reductions

3-SAT ≤p CLIQUE ≤p VERTEX-COVER ≤p SET-COVER

If you can solve B in polynomial time,
and A ≤p B (A reduces to B),
then you can solve A in polynomial time.

Cook-Levin Theorem: SAT is NP-Complete

Space Complexity

  • L: Logarithmic space
  • NL: Nondeterministic log space
  • PSPACE: Polynomial space
  • EXPTIME: Exponential time

Hierarchy: L ⊆ NL ⊆ P ⊆ NP ⊆ PSPACE ⊆ EXPTIME
