What is Computer Architecture?

Computer architecture is about understanding how computers work at the hardware level - the physical parts that make up a computer.

Simple Explanation: Just like a car has an engine, wheels, and steering, a computer has a CPU, memory, and storage that work together.

Main Components

  • CPU (Central Processing Unit): The brain of the computer that does calculations
  • RAM (Random Access Memory): Fast, temporary storage for running programs
  • Storage: Permanent storage like hard drives or SSDs
  • Input/Output: Keyboard, mouse, monitor, etc.

How the CPU Works

The CPU follows a simple cycle:

  1. Fetch: Get the next instruction from memory
  2. Decode: Understand what the instruction means
  3. Execute: Perform the instruction
  4. Store: Save the result

CPU Architecture and Design

Exploring instruction set architectures, pipelining, and memory hierarchy.

Instruction Set Architecture (ISA)

The ISA defines the interface between hardware and software:

  • RISC (Reduced Instruction Set Computer): Simple, fixed-length instructions (ARM, RISC-V)
  • CISC (Complex Instruction Set Computer): Complex, variable-length instructions (x86)

Pipelining

Pipelining allows multiple instructions to be processed simultaneously:

Classic 5-Stage Pipeline:
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  Fetch  │  Decode │ Execute │  Memory │  Write  │
│   (IF)  │   (ID)  │   (EX)  │   (MEM) │  (WB)   │
└─────────┴─────────┴─────────┴─────────┴─────────┘

Clock Cycle:  1    2    3    4    5    6    7
Instruction1: IF   ID   EX   MEM  WB
Instruction2:      IF   ID   EX   MEM  WB
Instruction3:           IF   ID   EX   MEM  WB

Memory Hierarchy

Speed vs. capacity tradeoff:

  • Registers: Fastest, smallest (nanoseconds)
  • L1 Cache: Very fast, small (1-2 cycles)
  • L2/L3 Cache: Fast, medium (10-50 cycles)
  • Main Memory (RAM): Slower, larger (100-300 cycles)
  • Storage (SSD/HDD): Slowest, largest (milliseconds)

Advanced Computer Architecture

Deep dive into superscalar execution, branch prediction, cache coherence, and multicore systems.

Superscalar and Out-of-Order Execution

Modern CPUs execute multiple instructions per cycle using:

  • Multiple Execution Units: ALUs, FPUs, load/store units
  • Register Renaming: Eliminates false dependencies
  • Reorder Buffer: Maintains program order for commits
  • Reservation Stations: Queue instructions for execution units

Branch Prediction

Branch Prediction Strategies:
├── Static Prediction
│   ├── Always taken
│   └── Backward taken, forward not taken
└── Dynamic Prediction
    ├── 1-bit predictor
    ├── 2-bit saturating counter
    ├── Correlating predictors (m,n)
    └── Tournament predictors

Modern CPUs achieve > 95% prediction accuracy

Cache Coherence Protocols

In multicore systems, MESI protocol maintains cache consistency:

  • Modified: Cache line is dirty, only copy
  • Exclusive: Clean, only copy
  • Shared: Clean, may exist in other caches
  • Invalid: Cache line not valid

Parallelism Types

  • ILP (Instruction Level): Pipelining, superscalar
  • TLP (Thread Level): Multithreading, multicore
  • DLP (Data Level): SIMD, vector processing
